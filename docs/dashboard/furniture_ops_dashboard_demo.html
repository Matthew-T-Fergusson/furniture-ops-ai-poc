Operational commentary: anomalies, trends, outstanding work

This sanitized dashboard is a portfolio/demo version. It demonstrates how an operator can review pipeline health, aged listings, margin outliers, write-offs, and work cues from one static HTML report without exposing private operational data.

Anomaly/action cues

Cue
Pending-sale hygiene: demo pending items without complete scheduling metadata should be followed up or moved back to active if the buyer is stale.
Listing age: items with high days-since-listed should get a markdown, photo refresh, relist, cross-post, bundle, or disposal decision.
Write-off visibility: disposed inventory appears as separate COGS so operators can distinguish true sold-item economics from operational loss/write-off decisions.

Work-item cues

Cue
Compliance/bookkeeping lane: keep cash-flow taxonomy and receipt-backed expense capture clean before relying on margin reports for decisions.
Listing execution lane: prioritize aged active listings and stale pending-sale rows because those are closest to cash conversion.
Automation maturity lane: keep dashboard generation repeatable with context export, operator-authored analysis, and CI smoke checks.
